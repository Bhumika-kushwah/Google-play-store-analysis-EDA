# 📊 Google Play Store EDA Analysis

## 🔍 Project Overview
This project focuses on performing Exploratory Data Analysis (EDA) on the Google Play Store dataset to uncover meaningful insights about app performance, user engagement, and market trends.

The goal is to transform raw, messy data into actionable insights using Python-based data analysis and visualization techniques.

---

## 📁 Dataset Information
- Source: Google Play Store dataset  
- Features:
  - App  
  - Category  
  - Rating  
  - Reviews  
  - Size  
  - Installs  
  - Price  
  - Content Rating  

- Data Type: Mixed (Numerical + Categorical)  
- Nature: Real-world dataset with missing values and inconsistencies  

---

## 🧹 Data Cleaning & Preprocessing
The dataset required extensive cleaning before analysis:

- Removed duplicate records  
- Handled missing values  
- Converted data types:
  - Installs → numeric  
  - Price → numeric  
  - Size → standardized format  
- Cleaned inconsistent entries (symbols, text formats)

💡 Clean data is essential for accurate analysis

---

## 📊 Exploratory Data Analysis

### 🔹 Univariate Analysis
- Distribution of Ratings  
- Category frequency  
- App size distribution  

### 🔹 Bivariate Analysis
- Category vs Installs  
- Price vs Rating  
- Size vs Installs  

### 🔹 Multivariate Analysis
- Combined impact of Category, Price, and Ratings on Installs  

---

## 📌 Key Insights

- Family category has the highest number of apps (~19%)  
- Majority of apps are free, very few paid apps  
- Most app ratings fall between 4.0 – 4.5  
- Most apps are targeted for “Everyone” audience  
- App installs vary significantly across categories  

---

## 💡 Key Learnings

- Data cleaning is not just a step — it's the foundation  
- A single incorrect datatype (like Reviews as string) can mislead analysis  
- Also realized how critical proper data cleaning is before analysis  
- EDA is not just about visualization — it’s about finding meaningful stories in data
